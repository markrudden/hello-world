About Testing Formulas

Formulas are used in PeopleFluent Talent Management to help calculate complex business rules in the following areas:

  • Plan Formulas and Functions (Performance and Compensation)

  • Alerts (Performance)

  • Reward Cycle Worksheet Row-level Formulas (Compensation)

  • Goal Functions (Goal Management)

  • Data Maps (Goal Management, Performance Management, Compensation [Bonus, Salary, and Stock], Reward Cycles [Bonus, Salary, Stock, and Total Compensation])

Once formulas are created, it is essential to test them to ensure they operate as required.

Testing formulas consists of the following steps:

  1. Define the appropriate formulas.

  2. Define and save unit tests.

  3. Test each formula individually, or validate the entire set of formulas

The validate operation primarily checks that the actual result matches the expected result by running the formulas using the supplied unit tests. After running a unit test, the Result column displays one of three possible alert icons to indicate unexpected results or problems.

Table: Unit Test Alert Icons

Icon

Description

Indicates that either there are no tests associated with a formula or function or the tests have not been executed.

Indicates that all of the tests that have been run for the formula or function have passed.

Indicates that one or more of the tests for the function have failed.

  1. Note: To help prevent errors caused by null values, use the null-safe functions (such as safeAdd) in your formulas as applicable. If you have created formulas with different conditional paths (such as if-then-else), you will need to test all conditions to make sure that the correct data type is returned in each case.

A Test Suite column and icon appears in those areas where unit tests can be executed. The results of the test are indicated by one of three icons that appear in the Result column, which can vary depending on the success or failure of the test. For more information, see Troubleshoot Formulas.
